# 0) 문제 정의와 기회

## 사용자 관찰 (원문 인용, 오탈자 일부 수정)

> "요즘 Instagram이나 댓글로, 혹은 LinkedIn 아웃바운드로 템플릿 같은 거 많이 준다, 혹은 정보 정리된 내용 준다 그러는데 그 과정에서 리드를 수집하는 툴이 없는 것 같다."

## 핵심 문제

콘텐츠(템플릿/정리글/가이드)를 배포할 때:
- 소비(읽기)는 되는데 전환(리드 수집)이 약함.
- 폼을 따로 떼면 맥락이 끊기고, 공유 문서만 뿌리면 데이터가 남지 않음.

### 문제의 구체적 시나리오

1. **맥락 단절 문제**
   - 콘텐츠와 폼이 분리되어 있으면 사용자의 주의가 분산됨
   - 콘텐츠를 읽는 흐름이 끊기면서 전환율이 낮아짐

2. **데이터 손실 문제**
   - 공유 문서만 배포하면 누가 읽었는지, 얼마나 관심이 있는지 알 수 없음
   - 마케팅/세일즈 활동의 ROI 측정이 어려움

3. **도구 부재 문제**
   - Notion 같은 콘텐츠 플랫폼과 리드 수집 도구가 통합되지 않음
   - 기존 폼 도구들은 콘텐츠와의 통합이 부족하여 사용자 경험이 나뉨

## 해결 컨셉 (원문 인용, 오탈자 일부 수정)

> "링크로 접속해서 노션의 일부 내용이 블라인드되어 있고, 리드를 설문 폼처럼 입력해야 그 블라인드들이 걷히는 구조."

### 핵심 아이디어

1. **블라인드 기반 게이팅**
   - 콘텐츠의 일부를 가림 처리하여 "보고 싶은" 욕구를 자극
   - 사용자가 폼을 입력하면 가림이 해제되어 콘텐츠를 볼 수 있게 함

2. **통합된 경험**
   - 콘텐츠와 폼이 같은 페이지에 있어 맥락이 유지됨
   - 사용자는 콘텐츠를 읽다가 자연스럽게 폼을 만나게 됨

3. **재방문 시 편의성**
   - 한 번 입력하면 세션/로컬스토리지/쿠키에 저장하여 재방문 시 다시 묻지 않음
   - 사용자 경험을 저해하지 않으면서 리드를 지속적으로 수집 가능

## 제품 한 줄 정의

**Notion 콘텐츠를 사이트처럼 보여주되, '일부 구간/요소'를 가림 처리하고, 리드 폼 입력 시 해당 가림을 해제하는 도구.**

## 시장 기회

### 대상 시장

1. **B2B 세일즈 팀**
   - LinkedIn 아웃바운드 마케팅을 하는 세일즈 팀
   - 템플릿/원페이저를 공유하면서 리드를 수집하고 싶은 세일즈 담당자

2. **마케팅 팀**
   - 콘텐츠 마케팅을 하는 마케팅 매니저
   - 캠페인 콘텐츠를 배포하면서 유입 → 리드 전환을 연결하고 싶은 마케터

3. **콘텐츠 크리에이터**
   - 전문 지식을 공유하는 크리에이터
   - 자료/정리글/체크리스트를 공유하면서 구독/리드를 확보하고 싶은 크리에이터

### 경쟁 우위

1. **Notion 네이티브 통합**
   - Notion을 이미 사용하는 팀에게 추가 도구 학습 비용이 없음
   - Notion의 풍부한 콘텐츠 기능을 그대로 활용 가능

2. **다양한 UX 패턴 지원**
   - 상단/하단 폼, 모달, Floating CTA, Inline Blur Gate 등 다양한 패턴 제공
   - 운영자가 콘텐츠 특성에 맞게 선택 가능

3. **비침투적 접근**
   - "게이트"가 아니라 "체크포인트" 개념으로 사용자 경험을 저해하지 않음
   - 재방문 시 자동 해제로 사용자 친화적

## 기대 효과

### 운영자(콘텐츠 크리에이터) 관점

- **전환율 향상**: 콘텐츠를 읽는 사용자 중 리드로 전환되는 비율 증가
- **데이터 수집**: 누가 콘텐츠에 관심을 보였는지 파악 가능
- **ROI 측정**: 콘텐츠 배포 활동의 효과를 정량적으로 측정 가능
- **운영 편의성**: 별도 도구 학습 없이 Notion 링크만으로 즉시 배포

### 사용자(콘텐츠 소비자) 관점

- **자연스러운 경험**: 콘텐츠를 읽는 흐름이 크게 방해받지 않음
- **빠른 해제**: 간단한 정보 입력만으로 즉시 콘텐츠 확인 가능
- **재방문 편의**: 한 번 입력하면 다음부터는 자동으로 열림
- **신뢰감**: 강제적이지 않고 선택 가능한 방식으로 느껴짐

## 구현 시 고려사항

1. **Notion API/임베드 제약**
   - Notion의 공개 공유 링크만 지원 (MVP)
   - iframe 임베드 방식 사용
   - Notion의 DOM 구조 변경에 취약할 수 있음

2. **브라우저 호환성**
   - 다양한 브라우저와 디바이스에서 동일한 경험 제공 필요
   - 모바일 환경에서의 사용성 최적화 필수

3. **개인정보 보호**
   - GDPR, 개인정보보호법 등 규정 준수
   - 사용자 동의 수집 및 관리 체계 필요

4. **성능**
   - Notion 콘텐츠 로딩 속도 최적화
   - 블라인드 처리 시 성능 저하 최소화
