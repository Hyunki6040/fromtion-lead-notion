# 2) 사용자 시나리오 / 페르소나

## 페르소나 정의

MVP의 핵심 사용자를 3가지 페르소나로 정의한다. 각 페르소나는 서로 다른 목적과 중요 지표를 가지고 있으며, 이를 바탕으로 기능 우선순위를 결정한다.

---

## 페르소나 A: LinkedIn 아웃바운드 세일즈 (Sales Representative)

### 기본 정보
- **역할**: B2B 세일즈 담당자 / 세일즈 매니저
- **주요 도구**: LinkedIn, 이메일, Notion (템플릿/원페이저 관리)
- **경험 수준**: 중급 ~ 고급 (도구 사용에 익숙함)

### 목적
**"템플릿/원페이저" 링크를 보낸 뒤 리드를 확보하고 "누가 관심 있었는지" 확인**

### 구체적 사용 시나리오

#### 시나리오 A-1: 콜드 아웃리치
1. LinkedIn에서 잠재 고객에게 메시지를 보냄
2. "우리 회사의 솔루션 원페이저가 있는데, 한번 봐주시겠어요?" 라고 하면서 링크 공유
3. 잠재 고객이 링크를 클릭하여 원페이저를 확인
4. 원페이저의 핵심 내용(가격/제안 등)이 블라인드되어 있음
5. 잠재 고객이 이메일과 회사명을 입력하여 해제
6. 세일즈 담당자는 콘솔에서 "누가 원페이저를 열었는지" 확인
7. 관심 있는 고객에게 후속 연락을 진행

#### 시나리오 A-2: 템플릿 공유
1. 세일즈 템플릿(예: "B2B 제안서 템플릿")을 Notion에 작성
2. FORMTION으로 링크 생성 및 배포
3. 상단에 간단한 폼 배치 ("템플릿 받으시려면 이메일만...")
4. 템플릿을 받은 사람들의 정보 수집
5. 템플릿 사용 여부와 후속 질문을 위한 리드 풀 확보

### 중요 지표
- **전환율**: 폼 제출률 (페이지 뷰 대비 폼 제출 비율)
- **리드 품질**: 회사명/직무 기입률 (얼마나 상세한 정보를 남겼는지)
- **후속 연락 가능성**: 유효 이메일 비율, 회사 도메인 비율

### 제품 기대사항
- LinkedIn 메시지에 링크를 바로 붙여넣기 쉽게
- 모바일에서도 잘 보여야 함 (LinkedIn 모바일 앱에서 링크 클릭 시)
- 리드 정보를 빠르게 확인할 수 있는 간단한 대시보드
- CSV 내보내기로 CRM에 쉽게 연동

### 불편함/페인 포인트
- 현재는 Notion 링크를 그냥 공유하면 "누가 봤는지" 모름
- 별도 폼 도구를 사용하면 맥락이 끊김
- 리드 정보를 수집하는 과정이 번거로움

## 페르소나 B: 마케팅 매니저 (Marketing Manager)

### 기본 정보
- **역할**: 콘텐츠 마케팅 / 디지털 마케팅 담당자
- **주요 도구**: Notion (콘텐츠 기획/작성), 이메일 마케팅 도구, 분석 도구
- **경험 수준**: 중급 ~ 고급

### 목적
**캠페인 콘텐츠를 배포하면서 유입 → 리드 전환 연결**

### 구체적 사용 시나리오

#### 시나리오 B-1: 가이드 콘텐츠 배포
1. "2024 마케팅 트렌드 가이드" 같은 긴 콘텐츠를 Notion에 작성
2. 소셜 미디어(Instagram, LinkedIn)에 링크 공유
3. 콘텐츠의 핵심 부분(예: 중간 섹션)을 블라인드 처리
4. 사용자가 콘텐츠를 읽다가 블라인드 구간에서 폼 제출
5. 마케팅 매니저는 유입자 중 리드로 전환된 비율 확인
6. 마케팅 수신 동의를 받은 사용자들에게 후속 콘텐츠 발송

#### 시나리오 B-2: 웨비나 자료 배포
1. 웨비나를 진행하고 자료를 Notion에 정리
2. 참석자들에게 "자세한 자료는 여기서 확인하세요" 라고 링크 공유
3. Entry 모달로 접속 즉시 리드 정보 수집
4. 웨비나 참석자 + 추가 관심자까지 리드 풀 확대
5. 이메일 마케팅 도구로 연동하여 세그먼트별 캠페인 진행

#### 시나리오 B-3: 랜딩 페이지 대체
1. 작은 캠페인을 위해 별도 랜딩 페이지 제작 비용이 아까움
2. Notion으로 간단한 랜딩 페이지 작성
3. FORMTION으로 리드 수집 기능 추가
4. 빠르고 저렴하게 캠페인 페이지 완성

### 중요 지표
- **유입 대비 리드**: 페이지 뷰 대비 리드 수집 비율
- **동의 확보율**: 마케팅 수신 동의 비율
- **콘텐츠 참여도**: 스크롤 깊이, 블라인드 해제율
- **캠페인 ROI**: 각 캠페인별 리드 수집 비용 대비 효과

### 제품 기대사항
- 다양한 UX 패턴으로 콘텐츠 특성에 맞게 선택
- UTM 파라미터 지원으로 트래픽 소스 추적
- Webhook으로 마케팅 자동화 도구와 연동
- 리드 정보를 실시간으로 확인할 수 있는 대시보드

### 불편함/페인 포인트
- 랜딩 페이지 제작 비용과 시간이 많이 듦
- 콘텐츠와 폼이 분리되어 전환율이 낮음
- 리드 정보를 여러 도구에서 관리해야 함

## 페르소나 C: 콘텐츠 크리에이터 (Content Creator)

### 기본 정보
- **역할**: 전문 지식을 공유하는 크리에이터 / 인플루언서
- **주요 도구**: Notion (콘텐츠 작성), 소셜 미디어 (Instagram, LinkedIn, YouTube 등)
- **경험 수준**: 초급 ~ 중급 (기술적 도구보다는 콘텐츠에 집중)

### 목적
**"자료/정리글/체크리스트"를 공유하면서 구독/리드 확보**

### 구체적 사용 시나리오

#### 시나리오 C-1: 체크리스트 공유
1. "스타트업 초기 체크리스트" 같은 유용한 자료를 Notion에 작성
2. Instagram 스토리나 게시물에 "링크 프로필 참고" 형태로 공유
3. Floating CTA 버튼으로 방해 없이 콘텐츠 제공
4. 관심 있는 사용자가 CTA 클릭 → 폼 제출 → 체크리스트 확인
5. 이메일 수집 및 구독자 확보

#### 시나리오 C-2: 무료 자료 배포
1. "2024 디자인 트렌드 정리" 같은 정리글 작성
2. 일부만 공개하고 나머지는 블라인드 처리
3. 하단 폼으로 "전체 자료 받기" 유도
4. 자료를 받은 사람들의 이메일 수집
5. 후속 콘텐츠나 프로모션 발송

#### 시나리오 C-3: 강의 자료 공유
1. 온라인 강의를 진행하고 보충 자료를 Notion에 정리
2. 강의 시청자들에게 링크 공유
3. Entry 모달로 강의 자료 접근 시 리드 정보 수집
4. 강의 후속 연락 및 추가 콘텐츠 발송

### 중요 지표
- **이메일 수집**: 총 수집된 이메일 수
- **재방문률**: 한 번 방문한 사용자가 다시 방문하는 비율
- **콘텐츠 인기도**: 각 콘텐츠별 리드 수집 수
- **구독자 전환율**: 방문자 대비 이메일 수집 비율

### 제품 기대사항
- 설정이 간단하고 직관적이어야 함 (기술적 지식이 적어도 사용 가능)
- 예쁜 디자인 (브랜드 이미지에 영향)
- 모바일 최적화 (Instagram 등 모바일 트래픽이 많음)
- 비용이 저렴해야 함 (초기 크리에이터는 예산이 제한적)

### 불편함/페인 포인트
- 기술적 도구 사용이 어려움
- 별도 랜딩 페이지 제작 비용이 부담됨
- 콘텐츠 공유와 리드 수집을 동시에 하는 도구가 없음

## 공통 사용 패턴 분석

### 3.1 공통 점
모든 페르소나가 공통적으로:
- **Notion을 이미 사용 중** (콘텐츠 작성 도구로)
- **콘텐츠를 공유하고 싶음** (템플릿/가이드/자료 등)
- **리드를 수집하고 싶음** (후속 연락/마케팅을 위해)
- **간단하고 빠르게 시작하고 싶음** (복잡한 설정은 원하지 않음)

### 3.2 차이점

| 항목 | 페르소나 A (세일즈) | 페르소나 B (마케팅) | 페르소나 C (크리에이터) |
|------|-------------------|-------------------|----------------------|
| **주요 트래픽 소스** | LinkedIn DM, 이메일 | 소셜 미디어, 광고 | Instagram, LinkedIn |
| **중요한 UX 패턴** | Entry 모달, Inline Blur | 다양한 패턴 조합 | Floating CTA, 하단 폼 |
| **리드 품질 요구도** | 높음 (회사/직무 필수) | 중간 (이메일 중심) | 낮음 (이메일만) |
| **기술적 숙련도** | 중간 ~ 높음 | 높음 | 낮음 ~ 중간 |
| **예산** | 중간 ~ 높음 | 높음 | 낮음 ~ 중간 |
| **분석 요구도** | 중간 (후속 연락 중심) | 높음 (캠페인 분석) | 낮음 (단순 수집) |

## MVP 우선순위 결정에 미치는 영향

### 4.1 필수 기능 (모든 페르소나 공통)
- Notion 임베드 및 블라인드 기능
- 기본 폼 (이메일 필수, 이름/회사/직무 선택)
- 간단한 리드 목록 및 CSV 내보내기
- 모바일 최적화

### 4.2 중요 기능 (특정 페르소나에게 필수)
- **Entry 모달**: 페르소나 A (세일즈)에게 중요
- **다양한 UX 패턴**: 페르소나 B (마케팅)에게 중요
- **간단한 설정 UI**: 페르소나 C (크리에이터)에게 중요
- **Webhook 연동**: 페르소나 B (마케팅)에게 중요

### 4.3 향후 확장 기능
- CRM 통합 (페르소나 A, B)
- 이메일 마케팅 도구 연동 (페르소나 B, C)
- 고급 분석 대시보드 (페르소나 B)
- 템플릿 라이브러리 (페르소나 C)

## 사용자 여정 (User Journey) 맵

### 5.1 페르소나 A (세일즈) 여정
1. **인지**: LinkedIn에서 잠재 고객에게 메시지 발송
2. **관심**: 잠재 고객이 링크 클릭
3. **진입**: Entry 모달로 즉시 폼 표시
4. **전환**: 이메일/회사명 입력 후 원페이저 확인
5. **후속**: 세일즈 담당자가 리드 정보 확인 및 연락

### 5.2 페르소나 B (마케팅) 여정
1. **기획**: 캠페인 콘텐츠 기획 및 Notion 작성
2. **설정**: FORMTION에서 블라인드 및 폼 설정
3. **배포**: 소셜 미디어에 링크 공유 및 UTM 파라미터 추가
4. **모니터링**: 리드 수집 현황 실시간 확인
5. **활용**: Webhook으로 마케팅 도구 연동 및 세그먼트별 캠페인

### 5.3 페르소나 C (크리에이터) 여정
1. **작성**: 유용한 자료를 Notion에 작성
2. **간단 설정**: FORMTION에서 Floating CTA만 선택
3. **공유**: Instagram 스토리에 링크 공유
4. **수집**: 관심 있는 팔로워들이 이메일 입력
5. **활용**: 구독자에게 후속 콘텐츠 발송

## 페인 포인트 및 해결 방안

### 6.1 공통 페인 포인트
- **"복잡한 설정이 두려움"** → 단순 설정 UI + 고급설정 숨김
- **"모바일에서 안 되는 것 같음"** → 모바일 최적화 필수 + 미리보기 기능
- **"리드 정보를 어디서 봐야 할지 모름"** → 간단한 대시보드 제공

### 6.2 페르소나별 페인 포인트
- **페르소나 A**: "CRM에 일일이 입력하는 게 번거로움" → CSV 내보내기 필수
- **페르소나 B**: "캠페인 효과를 측정하기 어려움" → UTM 지원 + 기본 분석
- **페르소나 C**: "기술적 도구 사용이 어려움" → 최대한 간단한 UI
