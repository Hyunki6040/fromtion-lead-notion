# 3) 핵심 UX 패턴 (필수 포함)

## 개요

MVP는 다양한 사용자 시나리오와 콘텐츠 특성에 맞게 선택할 수 있는 4가지 핵심 UX 패턴을 제공한다. 각 패턴은 서로 다른 강도와 사용자 경험을 제공하며, 운영자가 복수 선택하여 조합할 수 있다.

---

## 요구된 패턴 (원문 인용, 오탈자 일부 수정)

> "1) 상단과 하단에 리드 설문을 넣거나 2) 접속하자마자 모달이 뜨거나 3) CTA 버튼만 떠있거나 이런 옵션들이 있을 것 같아."

---

## 패턴 1: Top/Bottom Embedded Form (상단/하단 폼)

### 개념
콘텐츠의 상단 또는 하단에 폼을 임베드하여, 문서처럼 자연스럽게 리드 정보를 수집하는 패턴이다.

### 특징
- **강도**: 중간 (문서 흐름을 방해하지 않으면서도 명확하게 폼 제시)
- **사용자 경험**: 콘텐츠를 읽기 시작하거나 읽은 후 자연스럽게 폼을 만남
- **적합한 콘텐츠**: 긴 글, 가이드, 체크리스트 등

### 상세 명세

#### 1.1 상단 폼 (Top Form)
- **위치**: Notion 콘텐츠 임베드 영역 바로 위
- **용도**: "읽기 전에 먼저" 또는 "가볍게 시작" 메시징
- **ON/OFF**: 운영자가 독립적으로 켜거나 끌 수 있음
- **카피 설정**: 
  - Title (예: "이 템플릿, 바로 써도 됩니다.")
  - Sub-title (예: "짧게 입력하면 블러가 즉시 풀립니다.")
  - 각각 상단/하단 폼에 대해 독립적으로 설정 가능

#### 1.2 하단 폼 (Bottom Form)
- **위치**: Notion 콘텐츠 임베드 영역 바로 아래
- **용도**: "읽은 후 강하게" 또는 "더 많은 정보가 필요하면" 메시징
- **ON/OFF**: 운영자가 독립적으로 켜거나 끌 수 있음
- **카피 설정**: 상단 폼과 별도로 설정 가능

#### 1.3 조합 규칙
- 상단만 ON, 하단만 ON, 둘 다 ON, 둘 다 OFF 모두 가능
- 상단과 하단은 서로 다른 카피와 스타일을 가질 수 있음
- 상단은 "가볍게", 하단은 "강하게" 메시징 분리 권장

#### 1.4 시각적 디자인
- 배경: 화이트 또는 약간의 그레이 (#F7F7F8)
- 테두리: 얇은 테두리 (#E9E9EC) 또는 그림자
- 패딩: 충분한 여백으로 콘텐츠와 구분
- 반응형: 모바일에서는 전체 너비, 패딩 조정

#### 1.5 폼 필드
- 운영자가 설정한 필드들 표시
- 필수 필드: 이메일
- 선택 필드: 이름, 회사명, 직무, 한 줄 질문 등
- 동의 옵션: 마케팅 수신 동의, 개인정보 처리 동의

#### 1.6 제출 후 동작
- 제출 성공 시 블라인드 해제
- 폼은 유지하거나 숨길 수 있음 (운영자 설정, 기본은 유지)
- 성공 토스트 메시지 표시

## 패턴 2: Entry Modal Gate (접속 즉시 모달)

### 개념
페이지 진입 즉시 모달로 폼을 노출하여, 사용자가 콘텐츠를 보기 전에 먼저 리드 정보를 수집하는 패턴이다.

### 특징
- **강도**: 높음 (콘텐츠에 접근하기 전에 필수로 폼 제출)
- **사용자 경험**: 즉시 집중을 받지만, 방해가 될 수 있음
- **적합한 콘텐츠**: 중요도가 높은 콘텐츠, 템플릿, 원페이저 등

### 상세 명세

#### 2.1 모달 트리거
- **기본**: 페이지 로드 완료 후 즉시 표시
- **고급 설정**: N초 후 자동 오픈 (예: 1초, 3초, 5초)
- **애니메이션**: 페이드인 + 슬라이드 업 효과

#### 2.2 모달 디자인
- **오버레이**: 반투명 검정 배경 (rgba(0, 0, 0, 0.5))
- **모달 박스**: 
  - 배경: 화이트
  - 둥근 모서리: 16px
  - 그림자: 중간 강도
  - 최대 너비: 500px (모바일에서는 전체 너비에서 약간의 패딩)
  - 위치: 화면 중앙 (모바일에서는 하단 고정 또는 중앙)

#### 2.3 닫기 옵션
- **기본**: 닫기 버튼 없음 (강제적)
- **고급 설정**: "닫기 허용" 옵션
  - 닫기 허용 시: X 버튼 또는 "Not now" 버튼 표시
  - 닫기 미허용 시: 폼 제출 전까지 모달이 계속 표시

#### 2.4 미리보기 옵션 (고급설정)
- "Preview first" 버튼 제공 (콘텐츠 일부를 먼저 볼 수 있게)
- 미리보기 후 다시 모달로 돌아와서 폼 제출 유도

#### 2.5 모달 내용
- **Title**: 운영자 설정 (예: "이 콘텐츠는 체크인 후 열립니다.")
- **Body**: 간단한 설명 (예: "입력 후 바로 전체 내용이 열립니다.")
- **폼 필드**: 운영자가 설정한 필드들
- **버튼**:
  - Primary: "Unlock" (오렌지 배경)
  - Secondary (닫기 허용 시): "Not now" (투명 배경)
  - Tertiary (미리보기 허용 시): "Preview first" (링크 스타일)

#### 2.6 제출 후 동작
- 제출 성공 시 모달 닫기 + 블라인드 해제
- 성공 토스트 메시지 표시
- 재방문 시 모달 다시 표시하지 않음 (unlocked 상태)

#### 2.7 모바일 최적화
- 모바일에서는 하단에서 슬라이드 업하는 Drawer 형태도 고려
- 또는 전체 화면 모달로 표시
- 키보드가 올라올 때 모달 위치 조정

## 패턴 3: Floating CTA Button (CTA 버튼만)

### 개념
화면 우측 하단(모바일은 하단 고정)에 CTA 버튼을 배치하고, 클릭 시 폼 Drawer/Modal을 오픈하는 패턴이다.

### 특징
- **강도**: 낮음 (콘텐츠 흐름을 최대한 방해하지 않음)
- **사용자 경험**: 필요할 때만 클릭하여 폼 제출
- **적합한 콘텐츠**: 자유롭게 읽히길 원하는 콘텐츠, 블로그 글 등

### 상세 명세

#### 3.1 버튼 위치 및 디자인
- **PC**: 우측 하단, 화면에 고정 (position: fixed)
- **모바일**: 하단 중앙 또는 우측 하단 고정
- **디자인**: 
  - 북마크/탭 형태로 화면에 붙어있음
  - 마진 없이 화면 가장자리에 배치
  - 오렌지 배경 (#FF5A1F) + 흰 글자
  - 둥근 모서리: 상단만 둥글게 (북마크 느낌)
  - 그림자: 최소한 (border 강조)
  - 아이콘: 자물쇠 ❌, 진행 아이콘 (▶, →, ▮ 등) 사용

#### 3.2 버튼 라벨
- **Collapsed 상태**: "Unlock", "Open", "Continue" 등
- 운영자가 고급설정에서 라벨 커스터마이징 가능
- 아이콘 + 텍스트 또는 아이콘만 표시 가능

#### 3.3 클릭 시 동작
- 버튼 클릭 시 폼 Drawer 또는 Modal 오픈
- **Drawer (권장, 모바일)**: 하단에서 슬라이드 업
- **Modal (PC)**: 중앙 모달 표시
- Drawer/Modal에는 폼 필드와 제출 버튼 포함

#### 3.4 Drawer/Modal 내용
- **Title**: 운영자 설정 (예: "Unlock to continue")
- **Sub-title**: 간단한 설명
- **폼 필드**: 운영자가 설정한 필드들
- **버튼**: "Unlock" (Primary), "Close" (Secondary, Drawer에서)

#### 3.5 스크롤 동작
- 스크롤해도 버튼은 항상 보임 (fixed position)
- 모바일에서는 스크롤 다운 시 버튼이 약간 위로 올라갈 수 있음 (키보드 공간 확보)

#### 3.6 제출 후 동작
- 제출 성공 시 Drawer/Modal 닫기 + 블라인드 해제
- 버튼은 유지하거나 숨길 수 있음 (운영자 설정, 기본은 유지하되 "Unlocked" 배지 표시)
- 재방문 시 버튼 표시하지 않음 (unlocked 상태)

#### 3.7 모바일 고정 옵션 (고급설정)
- 모바일에서 하단 고정 여부를 설정할 수 있음
- 기본은 고정이지만, 운영자가 해제할 수 있음

## 패턴 4: Inline Blur Gate (인라인 블러 게이트)

### 개념
본문 중간의 특정 지점에서 콘텐츠 일부를 블러 처리하고, 폼을 인라인으로 삽입하여 사용자가 "지금 이걸 보려면"이라는 맥락에서 폼을 제출하도록 유도하는 패턴이다.

### 특징
- **강도**: 매우 높음 (콘텐츠 진행을 막고 폼 제출을 강제)
- **사용자 경험**: 콘텐츠를 읽다가 막히는 경험, 하지만 맥락이 명확함
- **적합한 콘텐츠**: 핵심 부분이 명확한 콘텐츠, 단계별 가이드 등

### 상세 명세

#### 4.1 블러 구간 설정
- **위치**: 운영자가 "첫 번째 블러 구간", "두 번째 블러 구간" 등으로 설정 (MVP에서는 1개만 지원, 구조는 확장 가능)
- **방식**: 
  - 섹션 단위 블러 (Section Blur): 지정된 영역 전체 블러
  - 프리뷰 + 블러 (Preview-Then-Blur): 상단 N줄만 보여주고 나머지 블러

#### 4.2 인라인 카드 디자인
- 블러 구간 위에 또는 중간에 카드 형태로 폼 삽입
- **배경**: 화이트 또는 약간의 투명도
- **테두리**: 얇은 테두리 또는 그림자
- **위치**: 블러 구간의 상단, 중간, 또는 하단 (운영자 설정)

#### 4.3 카드 내용
- **Title**: 운영자 설정 (예: "여기부터 핵심입니다.")
- **Sub-title**: 간단한 설명 (예: "이메일만 입력하면 바로 이어집니다.")
- **폼 필드**: 운영자가 설정한 필드들 (최소화 권장: 이메일만 또는 이메일+이름)
- **버튼**: "Unlock this section", "Continue reading" 등

#### 4.4 블러 처리
- CSS blur 필터 적용 (blur(10px) ~ blur(20px))
- 오버레이로 어둡게 처리 (rgba(0, 0, 0, 0.3))
- 그라데이션 효과 (위쪽 투명 → 아래쪽 진해짐)
- 운영자가 블러 강도 조절 가능 (약/중/강)

#### 4.5 사용자 경험
- 사용자가 스크롤하다가 블러 구간에 도달
- 블러 처리된 콘텐츠가 "무언가 있다"는 것을 암시
- 인라인 카드에서 폼 제출 유도
- 제출 전까지는 스크롤해도 블러 구간은 해제되지 않음

#### 4.6 제출 후 동작
- 제출 성공 시 블러 해제 + 인라인 카드 제거 또는 축소
- 성공 토스트 메시지 표시
- 콘텐츠가 선명하게 표시되어 계속 읽을 수 있음

#### 4.7 고급 설정
- 블러 구간 위치 세부 조정 (드래그로 오버레이 영역 재조정, MVP 후순위)
- 프리뷰 높이 조절 (N줄 또는 Npx)
- 여러 블러 구간 지원 (MVP에서는 1개, 추후 확장)

## 패턴 조합 규칙

### 5.1 조합 가능성

#### 권장 조합
- **Entry Modal + Inline Blur Gate**: 모달에서 첫 번째 유도, 인라인에서 두 번째 유도 (강한 전환)
- **Top/Bottom Form + Inline Blur Gate**: 상/하단은 약한 유도, 인라인은 강한 락
- **Floating CTA만 단독**: 가장 덜 방해하는 방식

#### 충돌 조합
- **Entry Modal + Top Form**: 동시에 표시하면 사용자 경험이 나쁨 → Entry Modal이 우선
- **여러 Inline Blur Gate**: MVP에서는 1개만 지원

### 5.2 우선순위 규칙
1. Entry Modal이 있으면 최우선 표시 (다른 패턴과 동시 표시 안 함)
2. Inline Blur Gate는 블러 구간에 도달했을 때 표시
3. Top/Bottom Form은 Entry Modal이 없을 때만 표시
4. Floating CTA는 항상 표시 가능 (다른 패턴과 공존)

### 5.3 "이미 해제된 사용자" 규칙
- unlocked 상태이면:
  - 블라인드 제거
  - 폼/모달은 숨김 (단, 운영자 설정 시 "구독 CTA"로 남길 수 있으나 MVP는 숨김이 기본)
  - Floating CTA 버튼도 숨김 또는 "Unlocked" 배지만 표시

## 패턴별 사용 가이드라인

### 6.1 언제 어떤 패턴을 사용할까?

#### Top/Bottom Form 사용 시기
- 콘텐츠가 길고 단계적으로 읽히는 경우
- 사용자에게 선택권을 주고 싶을 때
- "가볍게 시작" 또는 "읽은 후 강하게" 메시징이 적합할 때

#### Entry Modal 사용 시기
- 콘텐츠의 중요도가 매우 높을 때
- 리드 수집이 최우선 목표일 때
- 템플릿/원페이저 같이 핵심 정보가 처음부터 있을 때

#### Floating CTA 사용 시기
- 콘텐츠를 자유롭게 읽히길 원할 때
- 방해 없이 자연스러운 전환을 원할 때
- 블로그 글 같은 일반 콘텐츠에 적합

#### Inline Blur Gate 사용 시기
- 콘텐츠의 핵심 부분이 명확할 때
- 특정 구간에서만 강하게 유도하고 싶을 때
- 단계별 가이드나 체크리스트에 적합

### 6.2 패턴별 예상 전환율 (참고)
- **Entry Modal**: 높음 (50-70%, 강제적이지만)
- **Inline Blur Gate**: 높음 (40-60%, 맥락이 명확)
- **Top/Bottom Form**: 중간 (20-40%, 자연스러움)
- **Floating CTA**: 낮음 (10-30%, 선택적)

> 실제 전환율은 콘텐츠, 카피, 대상에 따라 크게 달라질 수 있음

## 구현 시 고려사항

### 7.1 성능
- 각 패턴이 콘텐츠 로딩에 영향을 주지 않아야 함
- 모달/Drawer는 지연 로딩 고려
- 블러 효과는 CSS로 처리하여 성능 최적화

### 7.2 접근성
- 키보드 네비게이션 지원
- 스크린 리더 지원 (ARIA 라벨)
- 모달의 focus trap 처리

### 7.3 모바일 최적화
- 터치 친화적인 버튼 크기
