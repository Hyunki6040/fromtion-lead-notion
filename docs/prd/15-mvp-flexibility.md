# 15) MVP 범위에서의 "자유도" 요약

## 개요

MVP에서 운영자가 선택할 수 있는 옵션들과 그 범위를 명확히 정리한다. "설정 자유도는 높이되, 복잡한 옵션은 고급설정으로 숨겨서 단순화"하는 원칙을 반영한다.

---

## 15.1 사용자(운영자)가 선택할 수 있는 것

### 15.1.1 UX 패턴 선택

#### 선택 항목
- **상단 폼 (Top Form)**: ON/OFF
- **하단 폼 (Bottom Form)**: ON/OFF
- **Entry 모달 (Entry Modal)**: ON/OFF
- **Floating CTA**: ON/OFF
- **Inline Blur Gate**: ON/OFF

#### 제약사항
- 최소 1개는 선택해야 함
- Entry Modal이 있으면 Top/Bottom Form은 표시되지 않음 (충돌 규칙)
- Inline Blur Gate는 MVP에서 1개만 지원

#### 자유도
- ✅ 복수 선택 가능 (충돌 제외)
- ✅ 각 패턴 독립적으로 ON/OFF
- ✅ 조합하여 다양한 사용자 경험 제공

---

### 15.1.2 폼 위치별 카피 설정

#### 선택 항목
- **상단 폼**: Title, Sub-title (독립 설정)
- **하단 폼**: Title, Sub-title (독립 설정)
- **모달/CTA/인라인**: 각각 Title/Sub-title 설정 (고급설정)

#### 제약사항
- 해당 폼이 ON인 경우에만 설정 가능
- 텍스트 길이 제한 (Title: 100자, Sub-title: 200자)

#### 자유도
- ✅ 상단과 하단을 서로 다르게 설정 가능
- ✅ 운영자가 원하는 메시지 작성 가능
- ✅ 브랜드 톤앤매너에 맞게 커스터마이징 가능

---

### 15.1.3 블라인드 방식 선택

#### 선택 항목
- **섹션 블러 (Section Blur)**: 간단하고 안정적
- **프리뷰 + 블러 (Preview-Then-Blur)**: 호기심 유발 (추천)
- **키워드 블랙아웃 (Keyword Blackout)**: 정교하지만 베타

#### 제약사항
- 키워드 블랙아웃은 베타 기능 (실패 시 폴백)
- MVP에서는 1개의 블러 구간만 지원

#### 자유도
- ✅ 블라인드 방식 선택 가능
- ✅ 블러 강도 조절 (약/중/강, 고급설정)
- ✅ 프리뷰 높이 조절 (고급설정)
- ⚠️ 키워드 블랙아웃은 실험적 (안정성 낮음)

---

### 15.1.4 블라인드 위치 설정

#### 선택 항목 (간단 모드)
- **초반만 공개**: 상단 20%만 보이고 나머지 블러
- **중간부터 잠금**: 상단 50%는 보이고 나머지 블러
- **하단 핵심만 잠금**: 상단 80%는 보이고 하단 20% 블러

#### 선택 항목 (고급 모드)
- 퍼센트 기반 직접 지정 (예: 상단 60% 이후 블러)
- 픽셀 기반 지정 (MVP 후순위)

#### 자유도
- ✅ 프리셋 선택 또는 직접 지정
- ✅ 블러 구간 위치 세밀하게 조절 가능
- ⚠️ MVP에서는 1개의 블러 구간만 지원

---

### 15.1.5 폼 필드 선택

#### 필수 필드
- **이메일**: 항상 포함 (변경 불가)

#### 선택 필드
- **이름**: ON/OFF
- **회사명**: ON/OFF
- **직무**: ON/OFF (객관식, 기본 옵션 제공)
- **한 줄 질문**: ON/OFF (라벨 커스터마이징 가능)

#### 자유도
- ✅ 필요한 필드만 선택
- ✅ 선택 필드는 최소화하여 전환율 최적화 가능
- ⚠️ 이메일은 필수 (변경 불가)

---

### 15.1.6 동의 옵션 설정

#### 선택 항목
- **개인정보 처리 동의**: ON/OFF, 필수/선택 토글
- **마케팅 수신 동의**: ON/OFF (기본: OFF, 선택)

#### 제약사항
- 개인정보 처리 동의는 기본 필수 (권장)
- 운영자가 선택으로 변경 가능 (법적 리스크 고지 필요)

#### 자유도
- ✅ 동의 옵션 ON/OFF
- ✅ 개인정보 처리 동의는 필수/선택 선택 가능
- ✅ GDPR 등 규정에 맞게 설정 가능

---

### 15.1.7 해제 유지 기간 설정

#### 선택 항목
- **세션만**: 브라우저 닫으면 해제
- **7일**: 7일간 유지
- **30일**: 30일간 유지 (기본)
- **90일**: 90일간 유지 (고급설정)

#### 자유도
- ✅ 사용자 경험에 맞게 기간 선택
- ✅ 짧게 하면 재확인 강화, 길게 하면 사용자 편의성 향상

---

### 15.1.8 모바일/PC 별도 설정 (고급설정)

#### 선택 항목
- **모바일/PC 별도 설정 사용**: ON/OFF
- 모바일 설정: UX 패턴, 폼 카피, 블라인드 설정
- PC 설정: UX 패턴, 폼 카피, 블라인드 설정

#### 자유도
- ✅ 모바일과 PC에서 다른 설정 적용 가능
- ✅ 디바이스별 최적화된 사용자 경험 제공

---

## 15.2 단순화 원칙

### 15.2.1 기본 화면 구성

#### 표시되는 항목
1. **Notion URL 입력**
2. **UX 패턴 체크박스** (5개 옵션)
3. **폼 필드 선택** (체크박스)
4. **상단/하단 폼 카피** (해당 폼이 ON인 경우만)
5. **동의 옵션 ON/OFF**
6. **해제 유지 방식** (드롭다운)

#### 숨겨진 항목 (고급설정)
- Inline Blur Gate 위치 세부 조정
- 블라인드 강도
- Preview 높이
- Entry Modal 세부 설정
- Floating CTA 세부 설정
- Keyword Blackout
- 저장 정책
- Webhook 설정
- 모바일/PC 별도 설정
- UTM 파라미터

---

### 15.2.2 단순 설정 vs 고급 설정 구분

#### 단순 설정 (기본 화면)
- **목적**: 대부분의 사용자가 필요한 핵심 설정
- **원칙**: 직관적이고 간단하게
- **설명**: 최소한의 설명만 제공

#### 고급 설정 (접어서 숨김)
- **목적**: 고급 사용자가 세밀하게 조절할 수 있는 옵션
- **원칙**: 필요한 사람만 펼쳐서 사용
- **설명**: 각 옵션에 대한 상세 설명 제공

---

### 15.2.3 프리셋 제공

#### 블라인드 위치 프리셋
- "초반만 공개", "중간부터 잠금", "하단 핵심만 잠금"
- 사용자가 직접 퍼센트를 계산할 필요 없음
- 고급 사용자는 직접 지정 가능

#### 블라인드 방식 추천
- "프리뷰 + 블러 (추천)" 표시
- 사용자에게 권장 옵션 안내
- 고급 사용자는 다른 방식 선택 가능

---

## 15.3 제약사항 및 한계

### 15.3.1 MVP에서 지원하지 않는 것

#### 플랫폼 제약
- ❌ Notion 외 문서/웹페이지 임베드
- ❌ 커스텀 도메인 실제 연결 (UI만 제공)

#### 폼 기능 제약
- ❌ 복잡한 폼 로직 (조건부 필드)
- ❌ 다단계 설문
- ❌ 파일 업로드
- ❌ 결제 기능

#### 권한 관리 제약
- ❌ 팀/워크스페이스 기능
- ❌ SSO
- ❌ 역할 기반 접근 제어

#### 분석 기능 제약
- ❌ 고급 분석 대시보드
- ❌ 커스텀 리포트
- ❌ A/B 테스팅

---

### 15.3.2 제한사항

#### 기술적 제한
- Inline Blur Gate: 1개만 지원 (구조는 확장 가능)
- Keyword Blackout: 베타 기능, 안정성 낮음
- 커스텀 도메인: UI만 제공, 실제 연결은 추후

#### 정책적 제한
- 이메일 필드는 필수 (변경 불가)
- 최소 1개의 UX 패턴 선택 필수
- Entry Modal + Top/Bottom Form 조합 불가

---

## 15.4 확장 가능성

### 15.4.1 향후 추가 가능한 것

#### 구조적으로 확장 가능
- 여러 Inline Blur Gate 지원 (현재는 1개, 구조는 확장 가능)
- 여러 블러 구간 지원
- 드래그로 블러 구간 직접 지정

#### 기능적으로 확장 가능
- 커스텀 도메인 실제 연결
- 고급 분석 대시보드
- CRM 통합
- 이메일 마케팅 도구 연동

---

### 15.4.2 확장성을 고려한 설계

#### 데이터 모델
- JSON 형태의 설정 저장 (유연한 확장)
- 새로운 필드 추가 시 기존 데이터와 호환

#### API 설계
- RESTful API 구조
- 버전 관리 가능한 구조

#### 컴포넌트 구조
- 모듈화된 컴포넌트
- 새로운 UX 패턴 추가 용이

---

## 15.5 자유도 vs 단순성 균형

### 15.5.1 핵심 원칙

1. **기본 사용자**: 단순 설정만으로도 충분한 가치 제공
2. **고급 사용자**: 필요한 옵션을 모두 조절 가능
3. **점진적 복잡도**: 기본 → 고급설정으로 점진적으로 복잡도 증가
4. **명확한 가이드**: 각 옵션에 대한 설명과 권장사항 제공

---

### 15.5.2 사용자 여정

```
[기본 사용자]
  ↓
단순 설정으로 빠르게 시작
  ↓
필요 시 고급설정 펼쳐서 사용
  ↓
점진적으로 기능 학습
```

```
[고급 사용자]
  ↓
고급설정 바로 펼치기
  ↓
세밀하게 조절
  ↓
최적화된 설정 완성
```

---

## 15.6 설정 가이드 제공

### 15.6.1 도움말

#### 툴팁
- 각 설정 항목에 "?" 아이콘
- 호버 시 간단한 설명 표시

#### 가이드 링크 (추후)
- "어떤 UX 패턴을 선택해야 할까요?" 가이드
- "블라인드 위치는 어떻게 설정하나요?" 가이드

---

### 15.6.2 권장사항

#### 추천 표시
- "프리뷰 + 블러 (추천)" 같이 권장 옵션 표시
- 각 옵션에 대한 간단한 설명 (장단점)

#### 기본값
- 대부분의 경우 적합한 기본값 제공
- 사용자가 변경하지 않아도 작동

---

## 15.7 최종 정리: 선택 가능한 옵션 요약

| 항목 | 선택 범위 | 제약사항 |
|------|----------|---------|
| **UX 패턴** | 5개 중 복수 선택 | 최소 1개, Entry Modal 충돌 규칙 |
| **폼 위치별 카피** | 상단/하단 독립 설정 | 해당 폼 ON인 경우만 |
| **블라인드 방식** | 3가지 중 선택 | 키워드 블랙아웃은 베타 |
| **블라인드 위치** | 프리셋 또는 직접 지정 | MVP는 1개 구간만 |
| **폼 필드** | 선택 필드 ON/OFF | 이메일은 필수 |
| **동의 옵션** | ON/OFF, 필수/선택 | 개인정보 동의 기본 필수 |
| **해제 유지 기간** | 세션/7일/30일/90일 | - |
| **모바일/PC 별도 설정** | ON/OFF (고급설정) | - |

---

## 15.8 사용자 유형별 권장 설정

### 15.8.1 초보자 (간단하게 시작)
- UX 패턴: Floating CTA만 또는 Top Form만
- 블라인드: 프리셋 선택 (초반만 공개)
- 폼 필드: 이메일만 (최소화)
- 고급설정: 사용하지 않음

### 15.8.2 중급자 (균형잡힌 설정)
- UX 패턴: Top/Bottom Form 조합
- 블라인드: 프리뷰 + 블러 (추천)
- 폼 필드: 이메일 + 이름 + 회사명
- 고급설정: 필요 시 사용

### 15.8.3 고급자 (세밀한 최적화)
- UX 패턴: 다양한 패턴 조합
- 블라인드: 직접 지정, 키워드 블랙아웃 시도
- 폼 필드: 모든 필드 활용
- 고급설정: 모든 옵션 활용
- 모바일/PC 별도 설정 사용
