# 17) 최종 요약 (제품이 제공하는 경험)

## 개요

FORMTION MVP가 제공하는 전체 경험을 요약하고, 핵심 가치를 명확히 정리한다.

---

## 17.1 제품 한 줄 정의

**Notion 콘텐츠를 사이트처럼 보여주되, '일부 구간/요소'를 가림 처리하고, 리드 폼 입력 시 해당 가림을 해제하는 도구.**

---

## 17.2 핵심 가치 제안

### 17.2.1 운영자(콘텐츠 크리에이터) 관점

#### 해결하는 문제
- 콘텐츠를 공유했는데 "누가 봤는지" 모름
- 폼을 따로 떼면 맥락이 끊김
- 공유 문서만 뿌리면 데이터가 남지 않음

#### 제공하는 가치
1. **통합된 경험**: 콘텐츠와 폼이 같은 페이지에 있어 맥락 유지
2. **간단한 설정**: Notion 링크 하나로 즉시 배포
3. **다양한 패턴**: 콘텐츠 특성에 맞게 UX 패턴 선택
4. **데이터 수집**: 누가 콘텐츠에 관심을 보였는지 파악
5. **재방문 편의**: 한 번 입력하면 자동으로 해제 상태 유지

---

### 17.2.2 사용자(콘텐츠 소비자) 관점

#### 제공하는 경험
1. **자연스러운 경험**: 콘텐츠를 읽는 흐름이 크게 방해받지 않음
2. **빠른 해제**: 간단한 정보 입력만으로 즉시 콘텐츠 확인 가능
3. **재방문 편의**: 한 번 입력하면 다음부터는 자동으로 열림
4. **신뢰감**: 강제적이지 않고 선택 가능한 방식으로 느껴짐

---

## 17.3 핵심 기능 요약

### 17.3.1 Notion 콘텐츠 표시
- Notion 공개 공유 링크를 iframe으로 임베드
- 콘텐츠가 정상적으로 로드되는지 프리뷰 제공

### 17.3.2 블라인드(가림) 기능
- 콘텐츠의 일부를 블라인드 처리
- 3가지 방식: Section Blur, Preview-Then-Blur, Keyword Blackout (베타)

### 17.3.3 리드 폼 제출 및 해제
- 리드 폼 제출 후 블라인드 해제
- 필수 필드: 이메일
- 선택 필드: 이름, 회사명, 직무, 한 줄 질문

### 17.3.4 상태 유지
- 재방문 시 다시 묻지 않도록 세션/로컬스토리지/쿠키로 해제 상태 유지
- 기본 30일간 유지

### 17.3.5 다양한 UX 패턴
- 상단 폼, 하단 폼, Entry 모달, Floating CTA, Inline Blur Gate
- 복수 선택 가능, 조합하여 사용

### 17.3.6 운영자 기능
- 프로젝트 생성 및 설정 (3-Step)
- 실시간 미리보기
- 리드 목록 및 CSV 내보내기
- Webhook 연동

---

## 17.4 사용자 여정 요약

### 17.4.1 운영자 여정
1. **프로젝트 생성**: Notion URL 입력 → UX 패턴 선택 → 블라인드 설정
2. **설정 조정**: 실시간 미리보기로 확인하며 설정 조정
3. **배포**: 공유 링크 생성 및 공유
4. **리드 수집**: 리드 목록에서 확인 및 CSV 내보내기

### 17.4.2 사용자 여정
1. **링크 클릭**: 공유된 링크 클릭
2. **콘텐츠 확인**: Notion 콘텐츠가 로드되고 일부가 블라인드 처리됨
3. **폼 제출**: 설정된 UX 패턴에 따라 폼 제출
4. **블라인드 해제**: 제출 성공 시 블라인드 해제, 콘텐츠 확인
5. **재방문**: 다음 방문 시 자동으로 해제 상태 유지

---

## 17.5 제품 철학 (원문 인용, 오탈자 일부 수정)

> "한 번 입력하면 그다음부터는 계속 session, localStorage, cookie에 기록되어서 남아있고… 정보 같은 것들을 뿌릴 때 좋을 것 같다."

### 핵심 철학
- **"가두는 게이트"가 아니라 "진행시키는 스텝"**
- 콘텐츠는 숨기지 않고, 흐름을 멈추지 않음
- 독자의 "관심"을 "신호"로 바꿈

## 17.6 MVP 범위 요약

### 17.6.1 포함되는 것
- ✅ Notion 공개 공유 링크 임베드
- ✅ 3가지 블라인드 방식 (Section Blur, Preview-Then-Blur, Keyword Blackout 베타)
- ✅ 5가지 UX 패턴 (상단/하단 폼, Entry 모달, Floating CTA, Inline Blur Gate)
- ✅ 리드 수집 및 저장
- ✅ 상태 유지 (localStorage + Cookie)
- ✅ 실시간 미리보기
- ✅ 리드 목록 및 CSV 내보내기
- ✅ Webhook 연동
- ✅ 모바일 반응형

### 17.6.2 제외되는 것
- ❌ Notion 외 문서/웹페이지 임베드
- ❌ 복잡한 폼 로직 (조건부 필드, 다단계 설문)
- ❌ 팀/워크스페이스 기능, SSO
- ❌ CRM 풀동기화
- ❌ 고급 분석 대시보드
- ❌ 커스텀 도메인 실제 연결 (UI만 제공)
- ❌ 이메일 인증
- ❌ 유료화 기능

## 17.7 성공 지표

### 17.7.1 핵심 지표
- **전환율**: form_submit / page_view (목표: 30% 이상)
- **Unlock 성공률**: unlock_success / form_submit (목표: 90% 이상)
- **리드 품질**: 회사명/직무 기입률 (목표: 60%/50% 이상)
- **재방문 해제 유지율**: returning_unlocked / returning_users (목표: 80% 이상)

## 17.11 최종 정리: FORMTION이 주는 경험

### 사용자(콘텐츠 소비자) 관점
- 사용자는 "뺏겼다"가 아니라 **"진행했다"**를 느낌
- 자연스러운 흐름 속에서 선택적으로 리드 정보 제공
- 재방문 시 편리한 경험

### 운영자(콘텐츠 크리에이터) 관점
- 운영자는 "리드를 뽑았다"가 아니라 **"대화를 만들었다"**를 느낌
- 간단한 설정으로 즉시 배포 가능
- 다양한 패턴으로 콘텐츠 특성에 맞게 최적화
